# MiWarp 竞品分析：Agent 展示与审查能力 (2026.02-05)

> 来源：Claude Code、Cursor、Codex CLI、Windsurf、Cline、GitHub Copilot 最近 3 个月的重大升级

---

## 一、最值得吸收的 10 个核心模式

### 1. 双信号状态指示器 (Claude Code)
**现状：** MiWarp 用单一颜色表示状态
**升级：** 颜色 + 图标形状同时传达两个维度
- 颜色 = 状态（运行中/需输入/空闲/完成/失败/停止）
- 形状 = 进程（活跃 `*` / 已退出 `·` / 循环休眠 `◈`）
- 效果：一眼区分"正在跑"和"跑完了但在等你"

### 2. Peek 面板 (Claude Code)
**现状：** 点击会话才看详情
**升级：** 按空格预览，不离开列表
- 显示最新输出、当前需求、打开的 PR
- 多选题显示为数字键快捷选项
- Tab 填入建议回复
- 效果：10 个会话的巡检从 5 分钟降到 30 秒

### 3. 检查点时间线 (Cursor + Cline)
**现状：** MiWarp 只有工具调用列表
**升级：** 自动在重大变更前创建检查点
- 嵌入聊天流的时间线，可预览/恢复
- Cline 用 Shadow Git（独立于项目 Git 的隐藏仓库）
- 恢复选项：仅恢复文件 / 仅恢复任务 / 两者都恢复
- 效果：用户敢让 agent 自主工作，因为随时能回退

### 4. 上下文窗口可视化 (Claude Code)
**现状：** 用户不知道 context 被什么占满了
**升级：** 彩色比例条显示 200K 上下文消耗
- 分段着色：系统提示 / 环境信息 / CLAUDE.md / 文件读取 / 工具输出
- Cursor 有独立的 Context Usage Breakdown 诊断面板
- 效果：用户能主动管理 context，避免溢出

### 5. 阶段化执行显示 (Cline + Codex)
**现状：** 工具调用平铺，没有结构
**升级：** 清晰的阶段划分
- **规划阶段** — 分析文件结构、AST、搜索
- **执行阶段** — 文件编辑、终端命令
- **审查阶段** — 结果汇总、可执行命令
- Codex 有 Exec Cell（语法高亮 + 时长 + 退出码）
- 效果：用户知道 agent 在哪个阶段，预期要多久

### 6. 实时摘要 (Claude Code + Windsurf)
**现状：** 显示原始工具输出
**升级：** 用快速模型生成一句话摘要
- Claude Code 用 Haiku 模型每 15 秒刷新摘要
- Windsurf 有专门的规划 agent 持续优化长期计划
- 效果：非技术用户也能理解 agent 在做什么

### 7. 转录视图模式 (Claude Code)
**现状：** 只有一种显示密度
**升级：** 三种模式 Ctrl+O 切换
- **普通** — 工具调用折叠为摘要，完整文本响应
- **详细** — 每个工具调用、文件读取、中间步骤
- **摘要** — 只有最终响应和所做的更改
- 效果：新手用摘要，专家用详细，日常用普通

### 8. Diff 统计指示器 + 行内评论 (Claude Code + Copilot)
**现状：** diff 只是展示
**升级：** `+12 -1` 可点击徽章 → 行内 diff 查看器
- 点击任意行打开评论框
- 一次性提交所有评论
- Claude 读取评论后生成新的 diff 供复审
- 严重性标签：红色(重要/阻止合并) / 黄色(琐碎) / 紫色(已有问题)
- 效果：代码审查从"看完再想"变成"边看边评"

### 9. 云端 Agent + 制品交付 (Cursor)
**现状：** 只有本地 agent
**升级：** 云端 agent 产出截图/视频/日志
- 用户可接管 agent 的桌面测试软件
- 交付 merge-ready PR + 制品
- 入口：Web / Desktop / Slack / GitHub / Linear / API
- 效果：agent 的工作成果可演示、可验证

### 10. 分层权限系统 (Claude Code + Cline + Codex)
**现状：** 简单的允许/拒绝
**升级：** 多层权限体系
- **工具级粒度** — 每个工具独立的自动批准设置
- **沙箱策略** — 只读 / 工作区写入 / 完全访问
- **模型驱动风险评估** — 模型标记 `requires_approval` 标志
- **Hook 系统** — PreToolUse 自定义 shell/HTTP/LLM 审批
- 效果：安全与效率的精确平衡

## 三、MiWarp 实施优先级

### P0 — 立即实施（用户感知最强）
1. **阶段化执行显示** — 规划/执行/审查三阶段 + 状态指示器
2. **双信号状态指示器** — 颜色 + 形状的组合
3. **转录视图模式** — 普通/详细/摘要三种密度
4. **实时摘要** — 工具调用折叠为一句话

### P1 — 短期实施（功能差异化）
5. **检查点时间线** — 可预览/恢复的时间线
6. **Diff 统计 + 行内评论** — 可点击的变更统计
7. **上下文窗口可视化** — 比例条显示 context 消耗
8. **Peek 面板** — 空格预览会话

### P2 — 中期实施（竞争优势）
9. **分层权限系统** — 工具级粒度 + 沙箱策略
10. **`/goal` 实时面板** — 长任务的运行时间/轮次/token
11. **成本追踪** — 每请求/每任务的 token 和费用
12. **多 Agent 平铺** — 并行 agent 的网格布局

### P3 — 长期愿景
13. **云端 Agent + 制品** — 截图/视频/日志交付
14. **远程桌面接管** — 测试 agent 工作成果
15. **Flow 感知** — 从隐式信号推断用户意图
