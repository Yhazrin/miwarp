## 4. Coordination procedure

1. **Before writing**: each agent runs `git grep <symbol>` to confirm
   the symbol isn't already in a frozen file.
2. **After writing**: each agent runs `git diff --stat` and reports the
   touched files. If any are not on the agent's allowlist, the commit is
   rejected by the integrator.
3. **Conflict resolution**: integrator runs `git diff` segment by segment.
   No `git checkout --ours/theirs` on shared files. Behavior must be
   preserved in the integrator's local test run.
4. **i18n merging**: integrator uses a single `i18n:check` run as the
   merge gate. If two branches added the same key with different values,
   the integrator chooses the more specific one and notifies the
   originating agent.

## 5. Branch protection

Each agent's branch is **read-only** to other agents. They communicate
through:

- this document (ownership)
- the runtime contract (`v1.0.9-runtime-contract.md`)
- the transaction contracts (`v1.0.9-transaction-contracts.md`)
- ADRs in `docs/adr/`

No direct edits to a peer's branch. If a peer needs a new contract, the
request goes through the integrator and is recorded as an ADR.
