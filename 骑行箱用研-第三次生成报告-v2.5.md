## 一、研究预检

### 1.1 调研目标

| 序号 | 商业问题 | 核心验证点 |
|------|---------|-----------|
| 1 | 用户画像 | 电动行李箱已购用户的人群特征、使用场景、购买动机 |

| 3 | 功能需求与痛点 | 当前产品哪些体验好/差？核心升级期待是什么？ |
| 4 | 品牌认知 | 用户对各品牌的认知和价格预期？小米进入的机会窗口？ |
| 5 | 价格测试 | 用户的价格敏感度和支付意愿？ |

### 1.2 已知 vs 未知

| 维度 | 产品团队已知 | 需要被发现 |
|------|------------|-----------|
| 用户画像 | 基本轮廓（商务出差人群） | 细分人群差异、不同场景行为模式 |
| 痛点 | 续航/重量是痛点 | 痛点背后的深层机制、优先级排序 |
| 品牌 | 小米有品牌势能 | 小米在电动行李箱品类的具体溢价空间 |
| 价格 | 预期在2000-3000区间 | 精确卡位、不同人群的价格分水岭 |
| 智能化 | 助力是方向 | 用户对"智能"的真实定义和期望层次 |

- **用研类型：** 立项前调研（产品定义验证）

- **时长：** 2小时44分钟
- **地域：** 北京（逐字稿），多地（对标报告）

## 五、洞察卡

### 洞察卡1：骑行姿态的"身份认知失调"

**洞察层次：** 深层

**表面：** 用户想要更舒适的骑行姿态
**中层：** 当前箱体高度不够，成人腿部蜷缩、坐垫过硬无靠背
**深层：** 电动行李箱的核心购买动机是"成年人的省力工具"，但当前设计让成年人骑上去像"大人坐小孩车"——不仅造成身体不适，更产生**身份认知失调**（"我在公共场合骑这个合适吗"），导致在酒店大堂、写字楼等有形象要求的场景主动放弃使用

**证据：**
- [E-001] "坐姿蜷缩是成人用户的核心痛点" — "我的腿是蜷着的。就跟半蹲似的" (说话人7)
- [E-002] "形象顾虑导致室内场景放弃使用" — "穿成正装我肯定不会骑了的" (说话人3)
- [E-003] "社交压力是室内骑行的隐形障碍" — "觉得你怎么上这来？" (说话人8)
- [E-035] "外观一体化是关键需求" — "像一个整体的行李箱...放外头感觉很low" (说话人3)

**决策影响：** 直接影响产品的"可用场景覆盖率"——如果成人用户在室内场景不骑，骑行功能的价值减半

**建议行动：**
1. 箱体高度优化：将骑行坐高从当前约45cm提升至50-55cm
2. 坐垫升级：增加软垫+可折叠靠背
3. 脚踏设计：前伸式脚踏，减少腿部蜷缩角度
4. 外观一体化：骑行结构收纳后"像正常箱子"

**风险评估：** 坐高增加可能导致重心升高，需与稳定性联合优化；靠背增加可能影响收纳体积

---

### 洞察卡2：续航焦虑 ≠ 续航不足——用户买的是"不用担心"

**洞察层次：** 深层

**表面：** 用户希望续航12-16km
**中层：** 担心虚标、冬季衰减、手机充电共用电量
**深层：** 典型的**损失厌恶**心理——用户实际骑行仅2-3km/次，但要求4-6倍安全冗余。用户买的不是"够用的续航"，而是"完全不用担心的续航"

**证据：**
- [E-006] "续航焦虑的核心是虚标恐惧" — "焦虑，因为怕它的虚电" (说话人4)
- [E-009] "实际骑行距离远低于期望续航" — "每天两三公里最多了" (说话人3)
- [E-008] "满负荷6km是底线续航" — "保底，最低我们可以接受多少？就6了" (群体共识)

**定量+定性交叉：**
- 实际需求：2-3km/次
- 期望续航：12-16km
- 安全冗余：4-6倍
- 底线：满负荷6km可接受，空箱6km不接受

**决策影响：** 续航是购买决策的"必备门槛"，低于底线直接不买

**建议行动：**
1. 续航标定策略：明确标注"满负荷XX公斤下续航XX公里"
2. 双规格SKU：6km轻量版（~7kg）+ 10km续航版（~7.5kg）
3. 电量管理：预留"最后一公里助力"保底电量
4. 冬季衰减宣传：提前教育用户冬季续航衰减预期

**风险评估：** 双规格SKU增加供应链复杂度；续航标定透明化可能暴露实际性能短板

**关联数据：** 500g重量换4km续航——用户明确接受此trade-off（逐字稿群体共识）

### 洞察卡3：稳定性是"体感安全感"工程

**洞察层次：** 中层

**表面：** 用户担心拐弯侧翻
**中层：** 三轮架构在急转弯时重心偏移，刹车距离约2米
**深层：** 用户的"稳定性"判断不是基于工程参数，而是基于**体感安全感**——"看着稳"比"参数稳"更重要

**证据：**
- [E-010] "急转弯时侧倾明显" — "没减速的时候一拐打转向拐弯啊...侧倾肯定有" (说话人4)
- [E-012] "稳定性是感知层面的不安全感" — "容易翻，多少感觉容易翻" (说话人6)
- [E-013] "刹车距离过长" — "刹车距离比较长...得悠个两米" (说话人4)

**分群差异：**

| 群体 | 稳定性痛点排序 | 核心场景 |
|------|-------------|---------|
| 三轮用户 | 排序第1 | 拐弯、刹车 |
| 多轮用户（4-6轮） | 排序较低 | 已通过轮数解决 |
| 有孩家庭 | 排序第1 | 孩子骑行安全 |

**决策影响：** 稳定性是痛点排序第一（6/7人），直接影响骑行功能使用意愿

**建议行动：**
1. 四轮底盘+双后驱：稳定性提升可量化
2. 刹车距离优化：从2米缩短至1米以内
3. 轮子材质强调："航空轮""合金实心"
4. 转弯辅助：电子差速系统

**风险评估：** 四轮方案可能增加重量和成本；小轮减震不如大轮，需平衡稳定性与舒适性

**反例：** 6轮用户满意度高，但有人担心"其中一个轱辘偏了就容易坏"

### 洞察卡4："智能"的正确定义是"省力"

**洞察层次：** 深层

**表面：** 用户想要更多智能功能
**中层：** 被"一车四用"等噱头吸引但实际不用
**深层：** 用户经历了**两阶段认知转变**：购买前被"多功能"吸引，使用后发现真正需要的是**省力相关**的智能（助力、跟随、防丢），而非**功能堆叠**的智能

**证据：**
- [E-018] "多功能噱头化导致购买后失望" — "一车四用...实用性并不强" (说话人8)
- [E-019] "功能闲置是常态" — "就那踏板我也用不上...也不会卸" (说话人8)
- [E-033] "助力功能是最高价值点" — "方便真的是科技改变生活" (说话人7)

**智能需求优先级：** 助力 > 跟随 > 防丢 > 氛围灯 > 显示屏

**决策影响：** 定义了"智能化"的产品方向——不做功能堆叠，做"省力"深度优化

**建议行动：**
1. 核心智能：推行助力（随速度/坡度/载重动态调节）
2. 第二梯队：自动跟随 + 防丢定位（蜂窝+GPS+蓝牙）
3. 情绪价值：氛围灯 + 仪表盘
4. 不做：踏板模式、可拆卸电动车等噱头功能

**风险评估：** 助力功能的技术实现难度较高；自动跟随在复杂环境中可靠性待验证

### 洞察卡5：小米在电动行李箱品类有"品牌溢价窗口"

**洞察层次：** 中层

**表面：** 用户愿意为小米品牌付更高价格
**中层：** 在电动行李箱领域，小米被视为"大品牌进入新领域"，而非"性价比品牌"
**深层：** 电动行李箱市场已被竞品教育过价格预期（≥2000元），小米的品牌势能+生态优势形成了独特的溢价空间

**证据：**
- [E-022] "小米品牌溢价高于追觅100-900元" — 小米3099-3990元 (概念测试)
- [E-023] "2000元是电动行李箱的信任底线" — "电动行李箱 2000 块钱以下，他就不太可信了" (主持人)
- 对标报告：10/12人愿意购买小米品牌

**建议行动：**
1. 定价卡位：2499元（划线价），实际成交价略低
2. 品牌叙事：强调"小米进入电动行李箱领域"的科技感
3. 生态联动：米家APP互联作为差异化卖点

**风险评估：** 品牌溢价窗口可能随市场教育而收窄；小米做箱包不专业的负面认知需通过产品品质消除

### 洞察卡6：多箱出行是被忽略的高频场景

**洞察层次：** 中层

**表面：** 用户想要小箱子拖大箱子
**中层：** 家庭出行几乎必然多箱，当前方案不稳（绳拖/挂钩晃荡）
**深层：** 电动行李箱的"省力"价值在多箱场景下被放大——优雅解决"一拖二"可显著扩大使用场景

**证据：**
- [E-024] "当前拖挂方案不稳" — "能不能就给它粘好了？" (说话人5)
- [E-025] "磁吸是用户自发提出的方案" — "磁吸，给它吸上" (说话人4)
- 对标报告：个人多箱旅行 3/12 (25%) 提及

**建议行动：**
1. 短期：设计硬连接拖挂装置（磁吸+卡扣）
2. 中期：配套非电动大箱（同品牌），形成套装
3. 长期：自动跟随技术

**风险评估：** 拖挂方案增加产品复杂度；配套大箱增加SKU管理成本

### 洞察卡7：情绪价值是超越功能的购买回报

**洞察层次：** 中层

**表面：** 用户对骑行功能满意
**中层：** "情绪价值还是挺高的"——购买时的兴奋感持续到使用中
**深层：** 电动行李箱不仅是"省力工具"，更是"科技身份符号"——在公共场合骑行带来的关注和新奇感，是传统行李箱无法提供的**情绪溢价**

**证据：**
- [E-026] "情绪价值超越功能预期" — "情绪价值还是挺高的...源于我在当时购买时候它的感受要强" (说话人3)

**风险评估：** 情绪价值随市场普及可能递减；需持续创新维持新鲜感

## 六、创新方向+销售建议闭环

### 方向1：容量-重量-骑行三角平衡

| 要素 | 内容 |
|------|------|
| **方向描述** | 三电小型化+新型材料减重+轮组形式优化，在20寸标准箱体内容纳≥28L，重量控制在7-7.5kg |
| **用户验证** | 重量不满意 10/12 (83%)；容量不足 7/12 (58%)；坐姿不适 5/12 (42%) |
| **销售建议** | 主打"轻装出行"场景，强调"20寸箱≥28L+仅7kg"的参数对比 |
| **风险评估** | 碳纤维方案成本高（逐字稿末尾讨论）；减重可能牺牲结构强度 |

### 方向2：生态+智能差异化

| 要素 | 内容 |
|------|------|
| **方向描述** | 聚焦省力智能：助力推行+自动跟随+防丢定位；米家APP互联+小爱语音控制 |
| **用户验证** | 智能功能不足 4/12 (33%)；助力功能体验后最高评价；米家APP互联是竞品无此能力 |
| **销售建议** | 主打"科技改变生活"场景，强调"不只是行李箱，是出行方式的升级" |
| **风险评估** | 自动跟随在复杂环境中可靠性待验证；米家互联需生态协同 |

### 方向3：简约外观+细节做工

| 要素 | 内容 |
|------|------|
| **方向描述** | 收纳后"像正常箱子"的外观一体化设计；哑光+金属感材质；拉链/锁扣/提把/拉杆顺滑度对标高端箱 |
| **用户验证** | 外观设计 8/12 (67%)；做工细节 6/12 (50%)；"像一个整体的行李箱"是关键需求 |
| **销售建议** | 主打"商务出行"场景，强调"低调科技感+高端品质" |
| **风险评估** | 外观一体化设计可能增加工程难度；高端做工提升成本 |

## 九、反例与局限

### 已检查的反例

1. **坐姿问题**：说话人7打了9分——部分用户容忍度高
2. **重量问题**：说话人5"沉一点好"——不能简单减重
3. **续航问题**：说话人3"够用"——少数用户不焦虑
4. **稳定性**：6轮用户无此痛点——多轮方案有效
5. **智能化**：说话人7"其他都非常满意"——部分用户已满足
6. **容量问题**：逐字稿仅2/7提及——样本偏商务单人出行

### ⚠️ 反直觉发现

| 发现 | 定性观察（逐字稿 7人） | 定量数据（对标报告 12人） | 差异说明 |
|------|---------------------|----------------------|---------|
| 重量满意度 | 4/7提及，部分"够用" | 10/12不满意 | 逐字稿偏商务男性，对标含更多女性和家庭用户 |
| 容量满意度 | 仅2/7提及 | 7/12提及容量不足 | 逐字稿偏商务单人出行，家庭用户需求更高 |
| 外观重要性 | 散见各讨论 | 8/12系统提及 | 逐字稿未系统编码外观偏好 |

### 研究局限

1. **样本量**：逐字稿仅7人（1场），对标报告12人（多场）——7人数据的代表性有限
2. **均为已购用户**：缺少潜在用户视角
3. **地域限制**：逐字稿仅北京，对标报告多地
4. **品牌偏差**：小米被明确提及，可能存在"讨好主持人"偏差
5. **概念测试局限**：手板体验与成品差异大，反馈可能偏乐观
6. **价格测量差异**：概念测试价格（2899-3990）≠ 预算预期（≤2500占67%）
